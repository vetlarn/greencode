Grunnlaget for grønn kode

Ren kode vil si at koden er ryddig, enkel å lese og lett å vedlikeholde. Ved å tenke på ren kode når man programmerer unngås det unødvendig kompleksitet, som igjen bidrar til å kunne forbedre programmet eller finne eventuelle feil raskere. Et ekstra pluss med ren kode er at det gir lavere ressursbruk, da effektiv kode gjerne bruker mindre minne og prosessorkraft.

La oss se på et enkelt eksempel i programmeringsspråket Javascript:

Før (Rotete og lite lesbar kode):

let validName = false;
if (name.length >= 3 && name.length <= 20) {
  validName = true;
}

Etter (renere og mer selvforklarende kode):

const isValidName = (name) => {
    return name.length >= 3 && name.length <= 20;
  };

I eksemplet kan vi se at funksjonen har fått et klart og tydelig navn, noe som gjør at det er lettere å forstå hva den faktisk gjør.

Tips for ren og grønnere kode:

  • Del opp kode i små, tydelige funksjoner
  • Gi variabler og funksjoner beskrivende navn
  • Unngå duplisering

Kort oppsummert:

Når koden er enkel og tydelig, blir det enklere å lage grønn kode, som igjen fører til mindre energibruk og lavere CO2-utslipp.
